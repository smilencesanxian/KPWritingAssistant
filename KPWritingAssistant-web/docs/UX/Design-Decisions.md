## ADR-001: 色彩系统选择

**状态**: 已接受 ✅

**背景**:
产品面向PET考生（11-15岁）及家长，需要传达专业、可信赖的教育品牌形象，同时保持年轻化。

**决策**:
选择蓝色系作为主色调，具体为 `#2D6EE0`（智慧蓝）。

**原因**:
1. **教育行业认知**: 蓝色在教育产品中广泛使用，用户已形成"专业、可信"的认知
2. **目标用户**: 蓝色对青少年友好，不会过于幼稚或严肃
3. **品牌差异化**: 避免使用过于常见的教育蓝（如钉钉蓝、企业微信蓝），选择稍亮的蓝色
4. **无障碍**: 蓝色在浅色背景上对比度良好，符合WCAG标准

**替代方案**:
- 绿色系（被否）: 过于像健康/环保产品
- 橙色系（被否）: 过于活泼，缺乏教育产品的稳重感
- 紫色系（被否）: 不够主流，可能让部分家长觉得"不正经"

**影响**:
- 所有页面主按钮使用 `--primary-500: #2D6EE0`
- 品牌标识、Icon、Loading动画统一使用蓝色

## ADR-002: 底部TabBar导航设计

**状态**: 已接受 ✅

**背景**:
需要设计小程序的主要导航结构，平衡功能完整性和使用频率。

**决策**:
采用4个Tab的底部导航：首页、批改、易错点、亮点库

**原因**:
1. **首页**: 必须，展示核心数据（统计、快速入口）
2. **批改**: 高频功能，独立Tab方便快速进入
3. **易错点**: 核心差异化功能，需要突出展示
4. **亮点库**: 积累功能，用户会频繁查看

**未选择的Tab**:
- "我的"页（被合并）: 设置功能低频，通过首页头像或更多入口进入

**替代方案**:
- 5个Tab（加"我的"）: 担心过于拥挤，且"我的"使用频率低
- 3个Tab（合并易错点和亮点库）: 两个功能都很重要，不宜合并

**影响**:
- TabBar配置4个入口
- "我的"页改为通过首页头像或设置入口进入

## ADR-003: 范文三级级别设计

**状态**: 已接受 ✅

**背景**:
PRD要求支持用户选择范文优化目标：合格/优秀/卓越。

**决策**:
在批改结果页使用分段控制器（Segmented Control）展示三级选择，点击后即时重新生成范文。

**原因**:
1. **即时反馈**: 用户可以看到不同级别的差异，帮助理解
2. **操作简单**: 点击即可切换，无需跳转页面
3. **符合认知**: 三级递进（合格→优秀→卓越）符合学习进阶逻辑
4. **保留亮点**: 每个级别都保留原文亮点，体现"顺势而为"的产品理念

**替代方案**:
- 下拉选择器（被否）: 隐藏了选项，用户不知道有哪些级别
- 滑块选择（被否）: 三级用滑块不精确，且难以展示级别名称
- 独立页面选择（被否）: 增加了操作步骤，打断用户流程

**影响**:
- 批改结果页增加 `LevelSelector` 组件
- AI需要支持三个级别的生成Prompt
- 需要处理级别切换时的Loading状态

## ADR-004: 易错点"小专题"设计

**状态**: 已接受 ✅

**背景**:
PRD要求当同一错误出现2次时，标记为易错点并提供知识点讲解。

**决策**:
将易错点设计为"小专题"形式，包含：知识点讲解、错误例句、练习题、生成练习纸。

**原因**:
1. **闭环学习**: 发现错误→学习知识点→练习巩固，形成完整学习闭环
2. **个性化**: 基于用户的真实错误例句讲解，更有针对性
3. **家长价值**: 家长可以看到系统的"教学"能力，增加付费意愿
4. **差异化**: 区别于普通的作文批改工具，有独特的教育价值

**技术实现**:
- 预设常见错误类型的知识点库（主谓一致、时态、介词等）
- 错误例句从用户历史作文中提取
- 练习题可从题库匹配或动态生成

**替代方案**:
- 仅文字提示（被否）: 教育价值不足
- 链接到外部资源（被否）: 打断体验，且无法保证质量
- AI实时生成讲解（备选V2）: MVP阶段使用预设库保证质量

**影响**:
- 需要准备知识点库内容
- 易错点详情页设计较复杂
- 增加练习题生成功能

## ADR-005: 字帖采用答题卡格式

**状态**: 已接受 ✅

**背景**:
产品核心卖点之一是将范文生成可打印的字帖，帮助练字。

**决策**:
字帖按照PET答题卡格式排版（横线格、固定行距、右侧留白）。

**原因**:
1. **真实场景**: PET考试使用答题卡，用户在真实答题卡上练习更有针对性
2. **家长认可**: 答题卡格式让家长感觉"专业"、"正规"
3. **卷面分**: 在真实格式上练字，直接提升考试卷面分
4. **差异化**: 普通字帖App不会针对PET答题卡专门设计

**技术实现**:
- 需要获取PET答题卡尺寸参数（行高、行数、页边距）
- PDF生成时模拟答题卡横线
- MVP阶段使用开源手写体，V2增加衡水体

**替代方案**:
- 普通横线纸（被否）: 缺乏针对性
- 方格纸（被否）: 英文作文不使用方格
- 空白纸（被否）: 无法帮助控制行距和大小

**影响**:
- 需要研究PET答题卡真实尺寸
- PDF生成逻辑较复杂
- 需要衡水体字体授权（V2）

## ADR-006: OCR失败时提供手动输入

**状态**: 已接受 ✅

**背景**:
OCR识别可能失败（图片模糊、手写难以辨认等），需要提供备选方案。

**决策**:
当OCR失败时，提供"重新拍摄"和"手动输入"两个选项。

**原因**:
1. **不阻断流程**: 用户仍然可以使用产品，不会因为OCR失败而放弃
2. **覆盖场景**: 有些用户可能更愿意直接打字（如在电脑上写好作文）
3. **质量保证**: 手动输入的文本更准确，批改效果更好

**交互设计**:
- OCR失败后Toast提示
- 提供两个按钮选项
- 手动输入页面简洁，专注输入

**替代方案**:
- 强制重拍（被否）: 用户体验差，可能流失用户
- 仅支持拍照（被否）: 限制了使用场景

**影响**:
- 增加手动输入页面
- 需要支持文本输入的批改流程

## ADR-007: 亮点库设计为"智能积累"

**状态**: 已接受 ✅

**背景**:
产品需要提取作文中的好词好句，帮助用户积累素材。

**决策**:
亮点自动提取，但用户可确认是否加入亮点库；在批改新作文时提示复用。

**原因**:
1. **自动化**: 减少用户操作，AI自动识别亮点
2. **可控性**: 用户可以选择不加入（避免错误内容混入）
3. **复用价值**: 提示用户在后续作文中使用，真正发挥积累的价值
4. **鼓励作用**: 孩子看到自己的好词好句被收集，有成就感

**技术实现**:
- AI提取亮点时给出置信度
- 高置信度自动加入，低置信度提示用户确认
- 使用向量检索匹配新作文场景

**替代方案**:
- 完全手动添加（被否）: 操作繁琐，用户可能不会用
- 完全自动（被否）: 可能混入不合适的句子

**影响**:
- 亮点库页面需要确认/删除功能
- AI需要支持亮点提取
- V2需要向量检索功能

## ADR-008: 使用 Font Awesome 图标

**状态**: 已接受 ✅

**背景**:
需要一套统一的图标系统。

**决策**:
使用 Font Awesome 6 Free 版本作为图标库。

**原因**:
1. **丰富性**: 提供免费图标数量多，覆盖常用场景
2. **一致性**: 所有图标风格统一
3. **易用性**: 通过CSS类名直接使用，支持大小、颜色调整
4. **性能**: 字体图标体积小于图片，支持矢量缩放

**微信小程序适配**:
小程序中Font Awesome使用受限，可选方案：
1. 使用图标图片（推荐）
2. 使用Vant等组件库图标
3. 引入图标字体（需配置）

**替代方案**:
- 自定义图标（被否）: 设计成本高
- 其他图标库（被否）: Font Awesome最主流，文档丰富

**影响**:
- 所有图标使用Font Awesome类名
- 小程序需要准备对应的图标图片

## ADR-009: 空状态使用插图+引导

**状态**: 已接受 ✅

**背景**:
用户首次使用或数据为空时，需要展示空状态。

**决策**:
空状态使用插图+标题+描述+引导按钮的组合。

**原因**:
1. **情感化**: 插图让空状态不单调，缓解用户焦虑
2. **引导性**: 明确告诉用户下一步该做什么
3. **品牌感**: 统一风格的插图增强品牌认知
4. **教育场景**: 青少年用户更喜欢可视化的引导

**设计规范**:
- 插图尺寸: 200-240px
- 标题: 18px, 600
- 描述: 14px, 400
- 引导按钮: 主按钮样式（如有必要）

**替代方案**:
- 纯文字（被否）: 过于单调
- 仅图标（被否）: 引导性不足

**影响**:
- 需要准备各场景的空状态插图
- 首页、亮点库、易错点、历史记录都需要空状态设计

## ADR-010: 每周提醒功能设计

**状态**: 已接受 ✅

**背景**:
PRD要求支持每周提醒生成易错题练习纸。

**决策**:
在"我的"页提供开关，开启后可选择提醒时间；每周自动生成PDF并推送。

**原因**:
1. **学习习惯**: 固定时间提醒帮助养成学习习惯
2. **个性化**: 基于用户真实错误生成，每次内容不同
3. **家长价值**: 家长可以看到系统的"主动教学"能力
4. **留存**: 定期推送提高用户留存和活跃度

**技术实现**:
- 使用微信小程序订阅消息
- 服务端定时任务生成PDF
- 用户可设置提醒时间（默认周一早上8点）

**替代方案**:
- 每日提醒（被否）: 频率过高，可能打扰用户
- 每月提醒（被否）: 频率过低，效果不佳
- 不自动推送，仅生成（被否）: 用户可能忘记查看

**影响**:
- 需要服务端定时任务
- 需要订阅消息权限申请
- "我的"页增加设置项

## ADR-011: 字体使用系统默认字体

**状态**: 已接受 ✅

**背景**:
需要确定产品使用的字体方案。

**决策**:
使用系统默认字体栈，不引入第三方字体文件。

**原因**:
1. **性能**: 无需下载字体文件，加快首屏加载
2. **兼容性**: 系统字体在各平台显示效果最佳
3. **体积**: 小程序包大小限制严格，避免字体文件占用
4. **维护**: 无需管理字体授权和更新

**字体栈**:
```css
/* 中文 */
font-family: -apple-system, "PingFang SC", "Hiragino Sans GB", "Microsoft YaHei", sans-serif;

/* 英文 */
font-family: -apple-system, "SF Pro Display", "SF Pro Text", "Helvetica Neue", sans-serif;
```

**例外**:
- 字帖PDF生成时需要使用手写体（Gochi Hand / 衡水体）

**替代方案**:
- 引入思源黑体（被否）: 增加包体积
- 使用商用字体（被否）: 授权成本高

**影响**:
- 所有文字使用系统字体
- 需要测试不同系统的显示效果

## ADR-012: 使用 rpx 进行尺寸适配

**状态**: 已接受 ✅

**背景**:
微信小程序需要适配不同屏幕尺寸。

**决策**:
使用 rpx 作为尺寸单位，基于 750rpx = 设计稿 375px 的换算。

**原因**:
1. **自动适配**: rpx 自动根据屏幕宽度缩放
2. **简单换算**: 设计稿 px 值 × 2 = rpx
3. **官方推荐**: 微信小程序官方推荐的适配方案

**换算规则**:
```
设计稿 375px 宽度下：
- 20px → 40rpx
- 16px → 32rpx
- 48px → 96rpx
```

**例外**:
- 边框线使用 px（避免模糊）
- 阴影模糊值使用 px

**替代方案**:
- 百分比布局（被否）: 不够精确
- vw/vh（被否）: 小程序支持但不常用
- 多倍图方案（被否）: 复杂度高

**影响**:
- 所有尺寸使用 rpx
- 开发时需进行单位换算
